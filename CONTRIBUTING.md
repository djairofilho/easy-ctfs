# Commit Pattern

Use Conventional Commits with this format:

`type(scope): short summary`

Allowed primary types in this repository:

- `feat`: new feature or functionality
- `fix`: bug fix or behavior correction
- `chore`: maintenance, setup, tooling, or non-feature tasks

Examples:

- `feat(poa): add CBC padding-oracle challenge endpoint`
- `fix(api): handle invalid auth cookie format`
- `chore(repo): add python .gitignore`

Guidelines:

- Write in imperative mood ("add", "fix", "update")
- Keep the subject concise
- Use scope when it improves clarity
